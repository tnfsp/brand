# 完整自動化工作流程

> 從靈感挖掘到內容發布的一鍵流程

---

## 🚀 核心流程：一個指令搞定

```bash
/inspire
```

### 自動執行的步驟：

```
Step 1: 抓取最新筆記
📡 npm run fetch:murmur
├─ 從 murmur.wilsonchao.com 抓取
└─ 儲存到 content-hub/murmur-notes/

Step 2: 讀取所有靈感來源
📚 讀取兩個來源
├─ 最新 Murmur 筆記（#murmur + #readwise）
└─ 234 個現有靈感檔案（personal-inspiration/）

Step 3: AI 智能分析
🤖 Inspiration Agent 分析
├─ 識別情感信號（困惑、感動、反思、觀察）
├─ 識別故事性片段（場景、對話、轉折）
├─ 識別可發展主題
└─ 避免與現有靈感重複

Step 4: 產出靈感報告
📊 生成結構化報告
├─ 🔥 高優先級點子（立即可用）
├─ ⭐ 中優先級點子（需補充素材）
└─ 💡 長期追蹤主題

Step 5: 自動整合到點子庫 ✨
💬 詢問 Wilson
├─ A) 全部加入 → 自動更新 ideas-pool.md
├─ B) 選擇性加入（例如：1, 3）
└─ C) 只查看報告

Step 6: 確認完成
✅ 點子已加入 ideas-pool.md
└─ 建議下一步行動
```

## 📋 完整生態系統

### 靈感挖掘階段
```
/inspire
├─ 自動抓取 Murmur 筆記
├─ 讀取 personal-inspiration（234 個靈感）

### 內容創作階段
```

├─ Blog-Writing Agent
├─ 參考 writing-style-database.md

```

```
/ideas （管理點子庫）
├─ 檢視所有點子

└─ 補充細節
```

## 🔄 典型使用場景

/inspire
```

**結果**：
1. 自動分析過去一週的 Murmur 筆記
2. 產出 3-5 個新點子
3. 選擇性加入點子庫
4. 準備好下週的創作素材

**頻率**：每週一次

## 💡 進階技巧

### 技巧 1：主題式挖掘

如果你最近特別關注某個主題（例如：職場溝通），可以：

1. 在 Murmur 中多記錄相關觀察
2. 執行 `/inspire`
3. Agent 會識別出這個重複出現的主題
4. 產出深度內容點子

### 技巧 2：定期回顧舊靈感

```bash
/ideas
→ 檢視「待使用」的點子
→ 選擇一個開始創作
```

有些點子可能當時素材不足，但現在累積了更多經驗，可以重新發展。

### 技巧 3：跨渠道內容再利用

```
IG 貼文（簡短、視覺化）
   ↓
擴寫成 Blog 文章（深度、完整論述）
   ↓
摘要成 Telegram 頻道貼文
   ↓
素材累積後製作 YouTube 影片
```

一個好的點子，可以跨多個渠道重複利用。
